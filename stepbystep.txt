----Backend----
1)Established Project Structure: Organized the backend using a professional and scalable architecture, 
                                 separating concerns into folders like models, schemas, api, and db.

2)Configured Database Connection: Set up the SQLite database engine and session management in app/db/session.py
                                  to handle data persistence.

3)Defined SQLAlchemy Models: Created the database schema in app/models/product.py, 
                             defining how the Product table should look (e.g., ID, Name, Price, Category, and Image).

4)Implemented Pydantic Schemas: Developed data validation layers in app/schemas/product.py to 
                                ensure consistent data formats for both API requests and responses.

5)Created API Routes: Developed a functional GET endpoint in app/api/v1/products.py to 
                      fetch and return the list of products from the database.

6)Setup Main Entry Point: Integrated all modules (routes, database initialization, and middleware) 
                          in app/main.py to serve as the application’s core launch point.

----Frontend----

1)Core Setup (The Engine)
    src/app/store.js: This is the "Brain" of the app. It connects and manages all the different logic parts (Slices) like Products and Cart in one central place.

    src/services/api.js: The "Bridge" to your backend. We set up axios here with your backend URL so we don’t have to type the full address in every file.

2. Feature Logic (Business Logic)
    src/features/products/productSlice.js: Manages the "State" of your products. It handles fetching data from the backend using AsyncThunk and tracks if the data is still loading or if there was an error.

    src/features/products/ProductList.jsx: A UI component that talks to Redux, gets the list of products, and displays them as beautiful cards on the screen.

3. User Actions (Shopping Cart)
    src/features/cart/cartSlice.js: Handles the logic for the cart. When a user clicks "Add to Cart," this file manages adding the item, increasing the quantity, or removing it.

    src/features/cart/Cart.jsx: The "Cart Page" design. It shows the user exactly what items they have picked to buy.

4. Supporting Folders
    src/components/common/: Stores small, reusable UI pieces like custom buttons, loading spinners, or alert boxes.

    src/components/layout/: Holds the common parts of the website that stay the same on every page, like the Navbar and Footer.

    src/theme/index.js: The place to customize ChakraUI colors, fonts, and global styles.

    .env: A secret file to store your API URL so it's not hardcoded directly into your logic.

*)Created Cart Slice (cartSlice.js):

    We set up a dedicated "Cart State" to manage shopping activities.

    We wrote the addToCart logic, which intelligently checks if an item is already in the cart. 
    If it exists, it only increases the Quantity; otherwise, it adds the item as a new entry. 
    This keeps the cart organized and professional.

*)Updated Global Store (store.js):

    We registered the cartReducer in our central Store alongside the product reducer.

    This allows the entire application to "know" exactly what is inside the user's cart at any time.

*)Connected UI with Action (ProductList.jsx):

    We added an "Add to Cart" button to every product card.

    Using the useDispatch hook, we connected these buttons to our Redux action so that clicking them 
    instantly sends the product data to the cart state.

*)Implemented Dynamic Navbar (Navbar.jsx):

    We created a navigation bar that stays at the top of the page.

    We used the useSelector hook to pull the cart data into the Navbar, 
    allowing us to display a live Cart Count (Badge) over the shopping icon.
    Now, whenever you add an item, the counter updates automatically without refreshing the page.

*)Created Cart Drawer UI (CartDrawer.jsx):

    Integrated a professional slide-out panel using Chakra UI’s Drawer component.

    This allows users to review their cart items instantly without leaving the main product page.

*)Implemented Dynamic Calculations:

    Added logic inside the Drawer to automatically calculate the Total Price by multiplying each item's 
    price by its quantity.

*)Added Decrease Quantity Logic (cartSlice.js):

    Created a decreaseQuantity function in Redux with conditional logic.

    Logic: If the quantity is greater than 1, it reduces the count by one. 
    If the quantity is exactly 1 and the user clicks minus, it removes the item from the cart entirely using splice.

*)Connected Drawer with Navbar:

    Linked the shopping cart icon in the Navbar to the Drawer using the useDisclosure hook.

*)Enhanced UI Interactivity:
